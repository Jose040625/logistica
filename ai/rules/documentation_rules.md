# Documentation Rules

1. **Formato Markdown Obligatorio:** Toda documentación debe redactarse en formato `.md` siguiendo el estándar GitHub Flavored Markdown (GFM).
2. **Claridad sobre tecnicismos:** Explicar el código asumiendo que el lector es un estudiante o un junior developer.
3. **Mapeo 1:1:** Toda ruta de API documentada en `docs/11_api_endpoints.md` debe existir y funcionar en el código base, y viceversa. No deben existir rutas fantasma.
